# 垃圾识别系统 —— 需求验收标准

> 版本: v1.0
> 日期: 2026/04/22

---

## 一、后端验收标准

### 1.1 功能验收

| 序号 | 验收项 | 验收标准 | 优先级 |
|-----|--------|---------|-------|
| B-01 | FastAPI 服务启动 | 运行 `uvicorn main:app` 无报错，控制台显示服务已启动 | P0 |
| B-02 | 模型加载 | 服务启动时加载 YOLO 模型，控制台显示 "Model loaded" | P0 |
| B-03 | `/predict` 接口 | POST 上传图片后返回 JSON，包含 `success`, `detections` 字段 | P0 |
| B-04 | 检测结果格式 | 每个检测项包含 `label`, `confidence`, `bbox` | P0 |
| B-05 | 无目标图片 | 上传不含垃圾的图片，返回 `detections: []` | P1 |
| B-06 | 错误处理 | 上传非图片文件，返回 400 错误码和友好提示 | P1 |
| B-07 | CORS 配置 | WPF 客户端能正常调用，不被浏览器/客户端拦截 | P0 |
| B-08 | API 文档 | 访问 `/docs` 能看到 Swagger UI 并在线测试 | P1 |

### 1.2 性能验收

| 序号 | 验收项 | 标准 | 测试方法 |
|-----|-------|------|---------|
| B-09 | 单图推理时间 (GPU) | ≤ 2 秒 | 上传 1920x1080 图片，记录响应时间 |
| B-10 | 单图推理时间 (CPU) | ≤ 10 秒 | 同上，关闭 CUDA |
| B-11 | 内存占用 | ≤ 4GB | 任务管理器观察 |
| B-12 | 并发支持 | 同时上传 2 张图片不崩溃 | 开两个终端同时请求 |

### 1.3 自测 Checklist

后端开发提交给前端前，必须完成以下自检：

- [ ] 服务能正常启动，无报错
- [ ] `/docs` 页面能打开，能看到接口文档
- [ ] 用 `/docs` 的 "Try it out" 上传 5 张不同图片，都能得到正确结果
- [ ] 上传一张非图片文件（如 .txt），能返回错误信息
- [ ] 连续调用 10 次 `/predict`，每次响应时间稳定
- [ ] 确认访问地址（含 IP 和端口），能跨机器访问

## 三、集成验收标准

### 3.1 端到端流程

| 序号 | 场景 | 预期结果 |
|-----|------|---------|
| I-01 | 正常识别 | 选择图片 → 点击识别 → 显示结果图和检测列表 |
| I-02 | 批量识别 | 选择文件夹 → 点击识别 → 逐张处理，进度条更新 |
| I-03 | 后端未启动 | 点击识别 → 提示"无法连接到识别服务" |
| I-04 | 大图片处理 | 上传 4K 图片 → 正常处理，不崩溃 |
| I-05 | 无目标图片 | 上传不含垃圾的图片 → 显示"未检测到目标" |
| I-06 | 网络超时 | 后端响应慢 → 显示超时提示，不卡死 |

### 3.2 兼容性验收

| 序号 | 验收项 | 标准 |
|-----|-------|------|
| I-07 | 图片格式 | 支持 jpg, jpeg, png, bmp, webp |
| I-08 | 视频格式 | 支持 mp4, avi, mkv, mov (视频功能) |
| I-09 | 操作系统 | Windows 10/11 |
| I-10 | .NET 版本 | .NET 8 |

## 四、验收流程

### 4.1 后端提交

1. 完成后端代码 (`main.py`)
2. 完成自测 checklist
3. 提供以下交付物：
   - 代码文件
   - 测试结果截图（`/docs` 页面 + 测试结果）
   - 访问地址（IP + 端口）

### 4.2 主人验收

1. 用后端提供的地址，在 WPF 中测试
2. 按集成验收标准逐项测试
3. 发现问题 → 反馈给后端开发 → 修复 → 重新验收

### 4.3 验收通过标准

- 所有 P0 项目必须全部通过
- P1 项目允许最多 1 个未通过（需记录并计划修复）
- P2 项目（TODO 项）不参与当前验收
